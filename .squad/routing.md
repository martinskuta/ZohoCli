# Routing Rules

| Signal | Agent | Why |
|--------|-------|-----|
| Architecture / scope / API design | Keaton | Lead sets direction |
| Auth flow / API client / timesheet models | Fenster | Backend specialist |
| Test coverage / edge cases / validation | Hockney | QA focus |
| Triaged issues (lead / multi-agent) | Keaton | Sync gate |
| Single-agent work | Fenster or Hockney | Per expertise |
| "Team, ..." or ambiguous | Keaton | Lead prioritizes |
